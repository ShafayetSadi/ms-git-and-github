## The Problem Without Git

Imagine two people editing the same project:

- Person A edits `index.html`
- Person B edits `index.html`
- One sends their file to the other
- Someone's work gets overwritten

This is how projects break.

<br><br><br><br><br>

## What Git Adds for Teams

Git solves the problems above by giving teams:

- **Parallel work** (everyone can work at the same time)
- **A shared history** (who did what, and when)
- **Safe merging** (combine changes without overwriting)
- **Accountability** (each change has a name)

Git is not just a backup tool.
It is a collaboration tool.

<br><br><br><br><br>

## The Basic Team Workflow

1) Everyone clones the project
2) Each person creates a branch
3) Work happens on the branch
4) A pull request merges it into `main`

This keeps `main` stable and clean.

<br><br><br><br><br>

## Vocabulary You Must Know

- **Remote**: a copy of the repo on GitHub
- **Origin**: the default remote name
- **Main**: the stable shared branch
- **Branch**: a safe place to change code

Once this vocabulary is clear, team Git feels easy.

<br><br><br><br><br>

## Summary

Teams need Git to work together safely.
Without Git, collaboration becomes file chaos.
With Git, changes are organized, reviewed, and tracked.
