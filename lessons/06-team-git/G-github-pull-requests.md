## What Is a Pull Request?

A Pull Request (PR) is a request to merge your branch into `main`.
It gives the team a chance to:

- Review changes
- Discuss improvements
- Run checks before merging

PRs are the standard workflow on GitHub.

<br><br><br><br><br>

## Basic Pull Request Flow

1) Push your branch to GitHub

```bash
git push -u origin feature/login-form
```

2) Open GitHub and click "Compare & pull request"
3) Write a clear title and description
4) Request reviews if needed
5) Merge when approved

<br><br><br><br><br>

## What to Write in a PR

Include:

- What changed
- Why it changed
- How to test

Example:

```
Title: Add login form layout

Summary:
- Added HTML structure for login
- Added basic styles

Test:
- Opened index.html in browser
```

<br><br><br><br><br>

## Review Etiquette

- Keep PRs small and focused
- Respond to comments politely
- Update the branch if changes are requested

Good PRs create trust on a team.

<br><br><br><br><br>

## Summary

Pull requests are the gate to `main`.
They make teamwork safer, cleaner, and more professional.
