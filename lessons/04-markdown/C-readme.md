## Why README Matters

The README is the first thing people see.
It answers:

- What is this project?
- How do I run it?
- How do I contribute?

A good README turns a random repo into a usable project.

<br><br><br><br><br>

## A Simple README Structure

1) Title and one-line summary
2) Installation
3) Usage
4) Contributing
5) License

Keep it short, but complete.

<br><br><br><br><br>

## Minimal Example

```
# CLI Guess Game

A tiny terminal game where you guess a number.

## Installation

```bash
python3 src/main.py
```

## Usage

```bash
python3 src/main.py
```

## Contributing

Pull requests welcome.

## License

MIT
```

<br><br><br><br><br>

## Tips

- Use clear headings
- Provide copy-paste commands
- Update the README when the project changes

Your README is part of the product.
