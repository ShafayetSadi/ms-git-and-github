## Markdown Is Everywhere

Once you learn Markdown, you will see it everywhere in development.

Common places:

- GitHub issues and pull requests
- Documentation sites
- Changelogs (`CHANGELOG.md`)
- Wikis and internal docs
- Static site generators
- Note apps and personal wikis

<br><br><br><br><br>

## Examples You Will Actually Use

- Writing a project roadmap in `docs/`
- Describing a bug in a GitHub issue
- Explaining a fix inside a PR description
- Creating a study note in Obsidian

Markdown is the default language for developer communication.

<br><br><br><br><br>

## Summary

Learning Markdown once gives you a tool for writing everywhere.
It is a small skill with a huge payoff.
