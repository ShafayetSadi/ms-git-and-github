## Why Etiquette Matters

Open source is a community effort.
Good code is important, but good communication is just as important.

<br><br><br><br><br>

## Read the Project Rules

Before contributing:

- Read the README
- Read CONTRIBUTING.md
- Follow the code style
- Use the right branch and commit rules

Every project is different.

<br><br><br><br><br>

## Be Respectful and Patient

Maintainers are often volunteers.
That means:

- Responses may take time
- Not every idea will be accepted
- Polite questions get better answers

Respect makes you memorable in a good way.

<br><br><br><br><br>

## Keep Scope Small

Large PRs are hard to review.
Start with small, focused changes.

If you want to make a big change, open an issue first.

<br><br><br><br><br>

## Credit and Licensing

Always respect the project license and credit original authors.
Do not copy code into a project without permission.

<br><br><br><br><br>

## Summary

Good etiquette makes open source pleasant and productive.
Be clear, be respectful, and follow the project rules.
