## Open Source Is a Community

Open source is not just "free code".
It is a community of people building software together.

Most projects have:

- **Maintainers** (make final decisions)
- **Contributors** (submit code or docs)
- **Users** (report bugs and request features)

Each role matters.

<br><br><br><br><br>

## How Work Usually Flows

1) Someone opens an issue
2) A contributor creates a branch
3) A pull request is opened
4) Maintainers review and merge

This process keeps quality high.

<br><br><br><br><br>

## The Reality (Important)

- Maintainers are often volunteers
- Not every PR gets accepted
- Response times can be slow

This is normal. Be patient and respectful.

<br><br><br><br><br>

## Licenses Matter

A license tells you what you can do with the code.
Common licenses:

- MIT
- Apache 2.0
- GPL

Always check the license before using or redistributing a project.

<br><br><br><br><br>

## Summary

Open source is about people and process.
Respect the workflow, communicate well, and you will succeed.
