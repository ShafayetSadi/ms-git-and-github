## Start Small

Your first contribution should be small and safe.
This builds confidence and trust.

Good first contributions:

- Fix typos
- Improve documentation
- Add examples
- Small bug fixes

<br><br><br><br><br>

## Where to Look on GitHub

Many projects label beginner-friendly issues:

- `good first issue`
- `help wanted`
- `beginner`
- `documentation`

On GitHub, you can search for these labels.

Example search:

```
label:"good first issue" language:javascript
```

<br><br><br><br><br>

## Before You Pick an Issue

1) Read the README
2) Read CONTRIBUTING.md
3) Check if the issue is active
4) Look at recent PRs

If the project looks inactive, choose another one.

<br><br><br><br><br>

## Ask Before You Start

Leave a short comment:

"Hi, I would like to work on this. Is it still open?"

This avoids duplicate work.

<br><br><br><br><br>

## Summary

Pick small issues, follow the rules, and communicate early.
That is the easiest path into open source.
